# File Naming Rules

## Purpose
Keep Project DinoCat Universe clean, searchable, and Codex-friendly.

## Rules
- Use English file names for system files.
- Use Markdown for documentation.
- Use lowercase folder names unless the file is a root-level governance file.
- Daily episode folders must use `day001`, `day002`, ..., `day365`.
- Do not store raw video exports directly in Git unless they are small review files.
- Large assets should live in external storage and be referenced in `assets/ASSET_INDEX.md`.

## Required Daily Folder Files
Each daily folder must contain:
- `Script.md`
- `Storyboard.md`
- `ImagePrompt.md`
- `VideoPrompt.md`
- `Voice.md`
- `Subtitle.md`
- `Publish.md`
- `Review.md`
- `ProductionLog.md`

## Versioning
Use semantic versioning:
- `v0.1.0` project foundation
- `v0.2.0` studio workflow foundation
- `v0.3.0` universe bible
- `v1.0.0` production-ready season system
